# Analysis: Missed PCNT Activity During COOLDOWN

## The Problem
The log shows the CPAP machine generating SD bus activity at 15:05:15 while the firmware was in the `COOLDOWN` state. However, when the firmware later transitioned to `LISTENING` at 15:05:51, the activity was ignored, and the FSM remained suppressed until a *new* activity event occurred at 15:11:23.

## Root Cause
The `TrafficMonitor` completely failed to detect the activity during `COOLDOWN` for two critical reasons:
1. **Hardware Disabled**: The FSM's `transitionTo()` function treated `COOLDOWN` as a "low-power" state and explicitly called `trafficMonitor.suspend()`. This stopped the PCNT hardware, released the APB clock lock, and engaged auto light-sleep. Any bus activity that occurred during this 1-minute window was completely ignored by the hardware.
2. **Software Blindspot**: The `loop()` function only called `trafficMonitor.update()` during `LISTENING` and `MONITORING`. Thus, even if the hardware caught a pulse, the software never evaluated `_lastSampleActive` during `COOLDOWN`.
3. **Hardware Purge**: Right before `COOLDOWN` transitions back to `LISTENING`, it explicitly calls `trafficMonitor.resetIdleTracking()`. This cleared the pulse counter, so any transient activity was wiped from memory before `handleListening()` ever got a chance to see it. 

We couldn't simply remove `resetIdleTracking()`, because the ESP32's *own* SD card commands (during `UPLOADING` and `RELEASING`) generate bus activity that triggers the PCNT hardware. 

## The Solution: The Activity Latch + Hardware Alive
To solve this reliably, we implemented a robust pipeline to safely bridge timing gaps between CPAP activity and FSM consumption:

1. **Hardware Alive**: `main.cpp` no longer suspends `TrafficMonitor` (or the APB clock lock) during `COOLDOWN`. The ESP32 consumes ~30mA instead of ~5mA during `COOLDOWN`, but it guarantees the hardware is awake to catch pulses.
2. **Latching**: We introduced a boolean `_activityLatch` into the `TrafficMonitor`. `loop()` now calls `trafficMonitor.update()` during `COOLDOWN` as well. Anytime `TrafficMonitor` detects PCNT pulses, it permanently sets `_activityLatch = true`.
3. **Pulse Preservation**: `resetIdleTracking()` was modified to read the hardware counter *before* clearing it. If any un-sampled pulses are present (drain > 0), it sets the `_activityLatch`.
4. **Selective Clearing**: We added `trafficMonitor.clearActivityLatch()` to `main.cpp`, calling it *exactly* at the moment the ESP32 finishes an upload sequence and enters `COOLDOWN`. This safely wipes the "false" activity generated by the ESP32's own SD card writes.
5. **Consumption**: Inside `handleListening()`, the FSM now checks `isBusy() || hasActivityLatch()`. If the latch was thrown during `COOLDOWN` (or any other state), `g_noWorkSuppressed` is immediately cleared and the latch is reset.

This guarantees that CPAP activity across *any* state is reliably captured and processed the moment the FSM is ready to evaluate suppression. The fix has been fully implemented.
